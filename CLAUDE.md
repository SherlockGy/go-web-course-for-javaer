# Go Web 快速学习教程 - 编写规范

## 核心原则

1. **使用最新 Go API**：Go 1.22+ 语法和标准库（如 `time.DateTime`、`http.Request.PathValue()`）
2. **与 Java 对比**：小而精的代码对比，解释设计理念差异，不是简单罗列语法
3. **最佳实践提醒**：用 📌 标记，解释为什么这样做，而非只说"应该这样"

---

## README.md 编写标准

### 内容要求

| 要素 | 要求 |
|------|------|
| **概念解释** | 该解释的地方适当解释，不要惜字如金变成纯文档 |
| **Java 对比** | 代码对比 + 设计理念差异 + 洞察总结 |
| **选择建议** | 提供"什么时候用什么"的决策表格 |
| **最佳实践** | 说明原因，避免空洞的"推荐/不推荐" |
| **版本说明** | 标注 Go 版本要求（如 Go 1.13+、Go 1.20+） |

### 避免的问题

- ❌ 过时描述（如"GOPATH 可选"应改为"现代 Go 不需要 GOPATH"）
- ❌ 夸张对比（如"秒级 vs 分钟级"需要解释原因）
- ❌ 未定义变量（如 `t.Format()` 但没解释 `t` 从哪来）
- ❌ 命令只列不解释（如 `go mod tidy` vs `go mod download` 的区别）
- ❌ 直接给作业答案（应该给提示，让学员自己完成）

### 结构模板

```markdown

## 学习目标

- 要求（不给答案）

```

## 示例代码编写标准

### 文件头注释

```go
// 示例名: 简短描述
//
// 📌 最佳实践:
//   - 要点1
//   - 要点2
//
// 🆚 与 Java 对比:
//   - Java: xxx
//   - Go: xxx
//
// 📌 运行: go run main.go
```

### 代码内注释

- 关键行用 `// 📌` 标记最佳实践
- 与 Java 差异处用 `// 🆚 Java:` 说明
- 新 API 标注版本：`// Go 1.20+`

### 每个示例必须有

- `go.mod` 文件（go 1.22）
- 可直接运行（`go run main.go`）
- 不依赖外部服务（数据库用 SQLite）

## homework 目录规范

### 必须提供模板文件

```go
// 作业：任务描述
//
// 📌 要求：
//   1. 要求1
//   2. 要求2
//
// 📌 提示：
//   - 关键 API 提示
//
// 📌 运行：go run main.go
package main

func main() {
    // TODO: 请完成
}
```

### 不直接给答案

- README 中的"提示"只给 API 用法，不给完整实现
- homework/main.go 用 TODO 标记需要填写的地方

## Review 检查清单

创建或修改章节时，检查以下项目：

### API 和语法
- [ ] 使用 Go 1.22+ 语法（如 `for i := range 10`）
- [ ] 使用最新标准库 API（如 `time.DateTime` 而非 `"2006-01-02 15:04:05"`）
- [ ] HTTP 路由使用 Go 1.22 新语法（如 `GET /users/{id}`）

### 内容质量
- [ ] 概念有足够解释，不是纯文档罗列
- [ ] 有 Java 对比和设计理念洞察
- [ ] 选择建议有决策依据（表格形式）
- [ ] 最佳实践说明了原因
- [ ] 无夸张或不准确的描述

### 结构完整性
- [ ] examples/ 目录有可运行示例
- [ ] homework/ 目录有模板（不是答案）
- [ ] 每个示例有 go.mod
- [ ] README 示例代码部分与实际目录对应

### 常见错误检查
- [ ] 无过时描述（GOPATH、旧 API）
- [ ] 代码示例中变量有来源说明
- [ ] 命令有使用场景解释
- [ ] 无直接暴露的作业答案
